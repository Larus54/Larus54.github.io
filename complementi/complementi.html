Pdf3 - Tipi di base

Esistono diversi tipi di oggetti presenti all’interno di python, partendo dai built-in:

  • Numerics: che si intendono la classe dei valori numerici (int, float, …)
  • Sequence (str, liste e tuple)
  • Set: fa parte degli insiemi
  • Dict: dizionari con coppia chiave valore

Per i tipi Numerics è possibile effetuare qualunque tipo di operazione aritmetica, usare diversi metodi presente come pow(), math.floor(), …

Per i tipi Sequence:

  • Le stringhe vengono rinchiuse nelle virgolette ed è possibile effettuare:

    • operazioni di accesso agli elementi come qualsiasi linguaggio(\(a[1]\))
    • effettuare delle concatenazioni con piu stringhe utilizzando il “+” (‘a’+‘b’ = ‘ab’)
    • operazioni di ripetizioni, per riperete sempre la stessa stringa piu volte (‘a’*4 = ‘aaaa’)
    • lunghezza tramite il metodo “len(<stringa>)”
    • Join: concatenare diversi elementi aggiungendo un separatore (';'.join([1,2,3]) = "1;2;3")
    • Split: Splittara una stringa tramite un separatore ('1+2+3+4+5'.split(+) = ['1','2','3','4','5'])

  • Le liste sono dei contenitori di elementi di qualsiasi tipo e vengono racchiuse tra le quadre '[]' e si possono utilizzare dei metodi gia all’interno di Python che possono renderle delle code (come lista.append(oggetto) e lista.pop(indice)) oppure degli stack (lista.append(oggetto) e lista.pop()). E’ possibile anche fare delle operazioni di Slicing cioè copiare da una lista di partenza una sottolista a scelta: (wt_slice = wt[1(start):3(stop)] -> [2,3]). Non viene utilizzato l’uguale tra due liste per fare la copiatura perchè si effettua copie per riferimento.

  • Le Tuple sono immutabili (cioè non modificabili) come le stringhe e vengono racchiuse all’interno delle tonde (es: tup1=('one', 'two', 21))

Per il tipo Set, si intende un insieme di oggetti non ordinati e non replicati (non può esistere piu di una volta l’oggetto).

Si creano con la funzione set() o {elem, elem}: a = set() o b=set([lista]) o c={1,2}

Vengono utilizzati per eliminare i duplicati all’interno da una lista e per i test di appartenenza

E si utilizzano i diversi tipi di cardinalità.

Per il tipo Dictionary, si intende un oggetto di associazione chiave valore per piu elementi e vengono racchiuse tramite graffe

Si crea tramite “{}” o dict(): a={}, b=dict() b={'chiave':'valore'}

Le chiavi all’interno del dizionario sono uniche.

I metodi piu importanti sono:

  • .keys(): tutta la lista delle chiavi all’interno del dict
  • .values(): tutta la lista dei valori all’interno di dict
  • del a[chiave]: elimina la chiave e il valore dal dizionario
  • .pop(chiave): elimina e restituisce il valore
  • .has_key(chiave): true se è presenta la chiave nel dizionario

Pdf5 - Funzioni

In python esistono le funzioni e si definiscono con la parola chiave def

La funzione rappresenta un blocco di codice:

def funzione(arg1, arg2):
    print(arg1)
    print(arg2)
    return 1

Ovviamente non si specificano nè i tipi dei parametri nè nel return.

def fib(n):
    a, b = 0, 1
    print(a)
    for i in range(n):
        print(b)
        a, b = b, a + b

E’ possibile definire dei parametri opzionali che assumono un valore di default se non vengono invocati

def fun(arg1, arg2=0):
...

Su python si può usare la notazione posizionale, quindi si intende che al richiamo della funzione si utilizzino le stesse posizioni degli argomenti della funzione (come su qualunque lnguaggio).

Ma si possono anche specificare a quale parametro ci si riferisce (keyword arguments)

a=fib(arg2=3, args1=1)

def fib(arg1, arg2):
    ...

Si può anche fare una fusione delle due metodi

Si utilizza l’operatore “*” anteposto ad un dizionario per ‘spacchettare’ sequenze e passare ogni elemento come notazione posizionale.

a=[1,2,3,4]

def fib(*a): -> fib(1,2,3,4)
    ...

Si utilizza l’operatore “**” anteposto ad un dizionario per ‘spacchettare’ ogni elemento come se fosse un keyword arguments

a={k1:3, k2:5}

def fib(**a): -> fib(k1=3, k2=5)
    ...

Gli operatori appena visti vengono anche utilizzati per la definizioni di due particolari parametri:

def fib(*args, **kwargs):
    ...

*args si intende tutti i parametri passati alla funzione vengono condensati all’interno di args.

**kwargs tutti gli arguments vengono condensati in un dizionario kwargs.

(utile se non si sa a priori il numero esatto dei parametri)

esempio:

# arg1 e arg2 non saranno presenti in args
def fib(arg1, arg2, *args, **kwargs):

Python è un linguaggio di ordine superiore, cioè che consente l’utilizzo di funzione come valori, infatti posso passare una funzione ad altre funzione e posso fare ritornare come risultato altre funzioni

def get_print_help_function(lang):
    def eng_help():
        print(‘help’)
    def ita_help():
        print(‘aiuto’)
    if lang == ‘eng’:
        return eng_help
    else:
        return ita_help

I valori di ritorno sono funzioni

Concetto di Closure

Supponiamo di voler avere una funzione inizializzabile che possa stampare sempre lo stesso messaggio quando viene chiamata

Prendiamo come parametro della funzione madre il messaggio poi nella funzione figlio stampiamo il messaggio.

def print_msg(msg):
    def printer():
        print(msg)
    return printer

#printer deve prendere "msg" per poter printare il messaggio.

Ed è proprio questo il concetto di CLOSURE, le funzioni innestate possono accedere alle
variabili delle funzioni madri.
Tali variabili vengono incapsulate nelle funzioni figlie in modo che possano essere utilizzate
anche quando la funzione madre termina.

Scope

Esistono diversi tipi di scope presenti in python.

  • Built-in: entità sempre disponibili, esempio: print
  • Global Scope: entità definite in blocco (non dentro a funzioni o blocchi)
  • Enclosing Scope: entità incapsulate per effetto della closure
  • Local Scope: entità definite dentro il blocco in cui si è
print('ciao') #built-in

a = 1 # global scope

def encaps():
    b=2  # b enclosing scope
    def local():
        c=3+b # local scope
        
        
def another():
    d = 0    # d non si vede da nessuno delle parti sopra

Il meccanismo di visione di python funziona cosi:

  • Si parte dal local
  • si guarda enclosed
  • poi global scope
  • infine il builtin

Chiamando le variabili con lo stesso nome si nasconde quelle degli scope piu esterno, si guarda lo scope piu interno e se solo non vi è si guarda quelli successivi.

Si può forzare l’utilizzo di un’entità dello scope globale con la parola chiave global

Gestione delle eccezioni

Le eccezioni vengono sollevate in due modi:

  • automaticamente: se si presentano anomalie in runtime per esempio una divisione per 0
  • manualmente: tramite il throw o il raise

Ovviamente c’è bisogno della sintassi per gestire le eccezioni:

    codice
except Classe_eccezione identificativo:
    codice

    codice
else:
    codice
finally:
    codice

All’interno delle funzioni si sollevano le eccezioni da riga di comando tramite il raise

Decoratori

Si usano i decoratori per aggiungere funzionalità ad una funzione (si intende quando dobbiamo utilizzare una funzione per piu risultati)

Ad esempio creare una funzione che cronometra il tempo di esecuzione di una funzione.
Farla per tutte le funzioni è lungo, di conseguenza si utilizzano i decoratori

# funzione per calcolare in funzione del tempo
# senza parametri

def time_function(function):
    def new_function():
        start = time.time()
        value = function()
        end = time.time()
        print ("ci ha messo ", end - start, " secondi")
        return value
    return new_function

F = time_function(fun) # passi una funzione per cui vuoi calcolare il tempo
F() # avvi time_function

Ovviamente la funzione può avere dei parametri (o X parametri)

# funzione per calcolare in funzione del tempo
# con parametri X

def time_function(function):
    def new_function(*args, **kwargs):
        start = time.time()
        value = function(*args, **kwargs)
        end = time.time()
        print ("ci ha messo ", end - start, " secondi")
        return value
    return new_function

F = time_function(fun) # passi una funzione 
# per cui vuoi calcolare il tempo
F(5,2) # avvi time_function

Per coprire tutti i casi possibili basta aggiungere con la sintassi “@”

@time_function
def func4(x,y,z):
    time.sleep(random.random()*x+(y-z))

Pdf 9 e 10 - Programmazione funzionale

Esistono diversi paradigmi di programmazione:

  • Imperativo: Il programma esegue istruzioni una dietro l’altra e il programmatore scrive le funzioni per risolvere il problema tipo C
  • Ad oggetti: Il programma è organizzato in oggetti che esibiscono metodi e incapsulano degli attributi che non vengono esibiti, come Java

Ne esistono degli altri, come il Funzionale e il Logico

Noi ci andremo a focalizzare sul Paradigma Funzionale:

E’ un tipo di paradigma in stile dichiarativo, ovvero si dichiara che valore deve assumere un dato nome.

Tipicamente un nome viene assegnato il risultato di una funzione

Si dice che utilizza “espressioni” come in matematica.

Le proprietà principali del paradigma funzionale sono:

  • Le funzioni utilizzate in questo paradigme vengono definite Pure: stesso input stesso risultato e non modificano gli argomenti in input
  • Non si usano cicli: ma la ricorsione
  • Sono linguaggi di ordini superiori: le funzioni posso essere passati come parametri di altre funzioni
  • I nome sono immutabili

E in python abbiamo tutti gli ingredienti per programmare in stile funzionale:

  • funzioni pure: Possibile in python basta non modificare in input e il risultato è sempre quello
  • no cicli: Possibile, usi la ricorsione in python
  • Ordine superiore: Si visto che si può fare in python tutto in oggetto.
  • I nomi sono immutabili: se non vengono cambiati rimangono quelli

esempio di paradigma funzionale

def fattoriale(x):
    return 1 if x<=1 else return x*fattoriale(x-1)
  • Stesso risultato in input non cambiano gli argomenti oppure input al problema
  • Ricorsione = no cicli
  • Oggetto funzione ritorna una funzione
  • immutabili, non vengono cambiati mai i nomi

NB: Utilizzare solo funzioni ricorsive può saturare lo stack, per questo python ha un limite di chiamate ricorsive, modificabile tramite:

import sys
sys.setrecursionlimit(<massimo_carattere>)

I vantaggi della programmazione funzionale sono:

  • Facilità di debug, il risultato dipende dagli argomenti
  • Funzioni piu semplici
  • Struttura piu interpretabile da un compilatore

Gli svantaggi della programmazione funzione sono:

  • Cambio di paradigma implica cambio di abitudini
  • Poco utilizzata: imperativo è piu semplice

Pdf12 - Gestione della memoria

I dati di un programma vengono salvati in memoria, a seconda del punto del programma essi vengono salvati in due zone differenti:

  • heap: memoria esplicatemente allocata (int *a = (int)malloc…)
  • stack: variabili dichiarate nel codice (int a = 10)

Le dimensioni e quando occuparla possono essere decise:

  • Manualmente: dal programmatore
  • Automaticamente: dal compilatore/interprete (garbage collectr, reference counting)

Per manuale intendiamo come in C l’utilizzo di diversi metodi come malloc() e free(), per avere una gestione esplicita e molto veloce, ma si rischiano molti memory leak e frequenti errori

Il reference couting è un algoritmo che libera memoria automaticamente quando un oggetto non è piu referenziato:

  • ogni oggetto ha un campo aggiuntivo che indica il numero effettivo di riferimento ad esso
  • appena il contatore arriva a 0, viene distrutto l’oggetto.

Questo algoritmo ha un problema di riferimenti circolari. Perchè due oggetti si possono referenziare a vicenda e anche dopo quando i due oggetti non sono piu referenziati altrove essi avranno comunque 1, di conseguenza non verrano mai distrutti.

Una possibile soluzione è quella di marcare come deboli i riferimenti ricorsivi e vengono considerati come tali per la rimozione:

  • li uso solo se almeno un oggetto è referenziato altrove
  • maggiore overhead

Il garbage collector è un entità che ciclicamente controlla lo stato degli oggetti in memoria e eventualmente li elimina (liberando memoria)

L’algoritmo che utilizza è il tracing:

  • Parte da degli oggetti radice segue tutti gli oggetti referenziati
  • Prosegue a cascata
  • Gli oggetti non referenziati vengono considerati garbage e eliminati

Per oggetti radice si intendo:

  • variabili globali
  • variabili locali
  • argomenti di funzioni

E non può piu essere raggiungibile per due motivi:

  • ragione sintattica: la sintassi indica che ormai quell’oggetto non esiste piu (utilizzata molto dal garbage collector)
  • ragione semantica: oggetti non piu raggiungibili per via del flusso di codice

E viene utilizzato quando un limite della memoria occupata raggiunge un limite.

Ogni oggetto viene associata una flag(mark and sweep)

  • 0 : irraggiungibile
  • 1 : raggiungibile

Il garbage effettua prima una fase di scansione dove applica i flag e successivamente una fase di deallocazione. (ovviamente i suoi limiti sono piu che evidenti, ogni volta è necessario scansionare tutto)

Il garbage collector utilizza anche un altro algoritmo chiamato Tri-color marking

  1. White: candidati per la rimozione
  2. Grey: oggetti raggiungibili ma i cui oggetti referenziati da essi non sono stati ancora analizzati
  3. Black: oggetti raggiungibili che non referenziano nessun oggetto white

Viene usato il flusso dal 1 al 3

Per liberare la memoria il garbage utilizza due strategie:

  • in movimento: copiare tutti gli oggetti raggiungibili in una nuova area di memoria
  • non in movimento: rilascio le zone di memoria dagli oggeti non raggiungibili

Python utilizza un approccio ibrido:

  • reference counting
  • garbage collector

il modulo gc consente di impostare il garbage collector, tramite gc.enable()/disable() ovviamente attivi il garbage.

La gestione della memoria in python dipende dall’interprete, il piu famoso CPython utilizza reference couting

import sys
a = 'aaa'
sys.getrefcount(a) -> 2

Esistono diversi programmi per profilare la memoria:

  • memory-profiler
  • tracemalloc

Essendo tutto un oggetto in python, l’interprete prende grandi aree di memoria dal SO e le rende private (heap privato) e le gestisce lui:

  • quando si crea un oggetto l’interprete lo alloca nell’heap privato
  • l’allocatore si occupa di creare in questa zona e gli assegna l’area di memoria
  • Se serve piu memoria chiama malloc() se è piu di 512 byte l’oggetto viene fatto direttamente malloc

Le zone di memoria di allocano in Aree e Pool

Ogni Area è divisa in pool per rendere piu efficace la memoria

Un pool ha la dimensione di una pagina stabilita da SO

Al suo interno ha dei blocchi tutti uguali.

Ogni blocco contiene solo un oggetto, appena viene liberato viene considerato libero
